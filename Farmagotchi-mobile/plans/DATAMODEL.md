# Farmagotchi Data Model Reference

This document outlines the core data models and schemas required for the Farmagotchi backend. It serves as a reference point for building the database schema (e.g., PostgreSQL, MongoDB) and API responses.

## Core Entities

### 1. User
Represents the farmer/account owner.
```typescript
interface User {
  id: string; // UUID
  email: string;
  name: string;
  location: {
    latitude: number;
    longitude: number;
    city?: string;
  }; // Used for weather insights
  createdAt: Date;
  updatedAt: Date;
}
```

### 2. Plot
Represents a specific physical field or crop area. A user can have multiple plots.
```typescript
interface Plot {
  id: string; // UUID
  userId: string; // Foreign Key -> User
  name: string; // e.g., "North Cornfield"
  cropType: string; // e.g., "Corn", "Tomatoes"
  sizeArea?: number; // Optional: Size of the field
  isActive: boolean;
  createdAt: Date;
  updatedAt: Date;
}
```

### 3. Farmagotchi (Pet)
The digital pet associated with a specific plot. There is a 1-to-1 relationship between a Plot and a Farmagotchi.
```typescript
interface Farmagotchi {
  id: string; // UUID
  plotId: string; // Foreign Key -> Plot (Unique constraint)
  name: string; // User-given name for the pet
  petType: string; // Defines the base species/visuals
  level: number; // Current level (e.g., 1-100)
  happiness: number; // Score from 0-100
  evolutionStage: 'EGG' | 'BABY' | 'JUVENILE' | 'ADULT';
  createdAt: Date;
  updatedAt: Date;
}
```

### 4. Task
A daily or recurring activity assigned to a specific plot.
```typescript
interface Task {
  id: string; // UUID
  plotId: string; // Foreign Key -> Plot
  title: string; // e.g., "Water the tomatoes"
  description?: string;
  frequency: 'DAILY' | 'WEEKLY' | 'ONCE';
  happinessReward: number; // Amount of happiness gained upon completion
  isCompleted: boolean; // Current status for the cycle
  lastCompletedAt?: Date; // To track daily resets
  createdAt: Date;
}
```

### 5. Scan (Crop AI Analysis)
Records of user-uploaded images and the resulting AI analysis for a plot.
```typescript
interface Scan {
  id: string; // UUID
  plotId: string; // Foreign Key -> Plot
  imageUrl: string; // URL to the stored image (S3, Cloud Storage)
  healthScore: number; // 0-100 overall health rating
  anomalies: string[]; // List of detected issues (e.g., ["Pest damage", "Nutrient deficiency"])
  tips: string[]; // Actionable advice generated by the AI
  happinessImpact: number; // The +/- change to the pet's happiness based on this scan
  createdAt: Date; // Timestamp of the scan
}
```

### 6. Transaction (Finance)
Financial records (income or expense) tied to a specific plot.
```typescript
interface Transaction {
  id: string; // UUID
  plotId: string; // Foreign Key -> Plot
  type: 'INCOME' | 'EXPENSE';
  amount: number; // Positive value
  category: string; // e.g., "Fertilizer", "Harvest Sale", "Equipment"
  notes?: string;
  transactionDate: Date; // User-defined date of transaction
  createdAt: Date;
}
```

### 7. Chat Message
The history of conversations between the user and their Farmagotchi regarding a specific plot.
```typescript
interface ChatMessage {
  id: string; // UUID
  plotId: string; // Foreign Key -> Plot
  sender: 'USER' | 'FARMAGOTCHI';
  content: string; // The text content of the message
  contextSnapshot?: object; // Optional: JSON storing the state (weather, happiness) at the time of the message
  createdAt: Date;
}
```

## Key Relationships & Business Rules
- **User -> Plots:** 1-to-Many.
- **Plot -> Farmagotchi:** 1-to-1. Deleting a plot should archive or delete the Farmagotchi.
- **Plot -> Tasks, Scans, Transactions, ChatMessages:** 1-to-Many. These are scoped strictly to the `plotId`.
- **Daily Resets:** The backend (or via local state sync) needs a mechanism to reset `isCompleted` on `Tasks` where `frequency` is `DAILY` based on the user's local timezone.
- **Happiness Bounds:** The `happiness` value on the `Farmagotchi` entity must always be constrained between `0` and `100`.
